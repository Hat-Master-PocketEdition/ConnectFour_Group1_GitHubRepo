DS - ADS Notes
-----------------
The structure for persistent data follows:

Within the file "GameHist.txt" (will push the file come development/plumbing)
The anatomy of the ADS of our history string is namecase first
Time "T", Win Condition "W", Game-type "G", Move Count "M", Color-schema "C".
The string will in theory be saved as:
MM,DD,YYYY,0,P,#,01

Delimiter is ','; Easy write and read. 
Win Condition is based on 0 - Player 1;
                          1 - Player 2;
                          2 - AI;
                          3 - Tie;

Game-type are "P" for Player vs. Player, "C" for Player vs. Computer.
Move Count is integer based, and will be used to display for the matter of "how many turns the game took".
Color Schema is a str, that defines how the colors are assigned via integer reference in the string.
The colors are Red: 0; Yellow: 1;
Final point reference C is that it is order of player color
"01" 0 - Player 1 Red; 1 - Player 2 Yellow;
In parsing, it allows easy determining of who won, and what color was assigned.

The only final piece would be to recreate the board, but that would be possibly too much memory reference, but could keep the state within
a separate GameState file, but that may be too ambitious to project at hand.

When we make the ListBox for the Statistics page, I will append it in singleline, and make a for loop to parse the ADS.
I will also count player1 victories (since Player 2 is N player, and Project README states do not multiple)
Then the accord becomes Computer victories, counting all the victories + total matches (line counter)
Thus preparing the ratios that are required for statistic display.
Percentages required are Computer Victories, and Player Victories.

Ratios for Reference:
(Victories / Game Total) * 100 = Victory Percentage;
